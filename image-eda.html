ImageEDA

The complete image dataset analysis module. This page explains how to use every important function, what each output means, and how to interpret the dashboards produced from a cat and dog dataset.

Image inventoryDataset qualityColour analysisTexture & frequencyDuplicate detection
Use ImageEDA when you want to understand an image dataset before model training: file validity, image sizes, label balance, brightness, sharpness, colour distribution, texture, duplicate images, and normalization statistics.

Recommended workflow

from pathlib import Path
from viseda import ImageEDA

output_dir = Path("image_eda_outputs")
output_dir.mkdir(exist_ok=True)

eda = ImageEDA(
    verbose=True,
    n_colors=8,
    phash_threshold=10,
    blur_threshold=50.0,
    compute_glcm=True,
    compute_freq=True,
)

eda.load("C:/Users/Ike/Desktop/testvis/", label_from_parent=True)
summary = eda.summary()

print(summary["inventory"])
print(summary["quality"]["brightness"])
print(summary["labels"])

eda.plot_colour(save_path=str(output_dir / "colour_dashboard.png"))
eda.plot_texture(save_path=str(output_dir / "texture_dashboard.png"))
eda.plot_samples(n=25, cols=5, save_path=str(output_dir / "samples.png"))
eda.report(str(output_dir / "image_eda_report.html"))

Constructor: ImageEDA(...)

ParameterMeaningPractical guidance
verbosePrints loading and processing progress.Set to True while testing; use False in automated scripts.
max_imagesLimits how many images are analysed.Useful for quick previews on very large datasets.
n_colorsNumber of clusters used for dominant colour extraction.Use 5–10 for normal datasets; higher values give more detailed palettes.
phash_thresholdHamming-distance threshold for near-duplicate grouping.Lower values are stricter; higher values catch more visually similar images.
blur_thresholdSharpness threshold used to flag blurry images.Increase it if you want stricter blur detection.
compute_glcmEnables GLCM texture features.Keep enabled for surface/texture-heavy datasets; disable for faster runs.
compute_freqEnables FFT frequency analysis.Useful for noise, detail, and image-frequency inspection.

Loading functions

load()

Loads images from a folder, one file, or a list of file paths. It supports common formats including JPG, PNG, BMP, TIFF, WEBP, and GIF. When label_from_parent=True, labels are taken from folder names such as dataset/Cat/img001.jpg and dataset/Dog/img002.jpg.

load_arrays()

Loads images that already exist as NumPy arrays. This is useful when images are generated, augmented, or loaded from another data pipeline instead of from disk.

# Folder-based loading
eda.load("C:/Users/Ike/Desktop/testvis/", label_from_parent=True, recursive=True)

# Array-based loading
import numpy as np
arrays = [np.random.randint(0, 255, (256, 256, 3), dtype=np.uint8) for _ in range(10)]
labels = ["no damage", "minor", "moderate", "heavy", "no damage", "minor", "moderate", "heavy", "no damage", "minor"]
eda.load_arrays(arrays, labels=labels)

Analysis functions

FunctionWhat it doesHow users should interpret it
summary()Returns a nested dictionary containing all computed dataset statistics.This is the main programmatic output. Use it in notebooks, logs, reports, or automated data-quality checks.
normalization_stats()Returns per-channel RGB mean and standard deviation in the [0, 1] range.Use these values in PyTorch/TensorFlow normalization so model inputs match the dataset distribution.
report(output_path)Creates a self-contained HTML report from the summary.Use this when sharing EDA results with supervisors, collaborators, or project teams.
summary = eda.summary()
print(summary["inventory"]["total"])
print(summary["spatial"]["height"])
print(summary["quality"]["sharpness"])

stats = eda.normalization_stats()
print(stats)
# Example use in torchvision:
# transforms.Normalize(mean=stats["mean"], std=stats["std"])

Summary dictionary explained

SectionMeaningWhy it matters
inventoryTotal, valid, and corrupt images; file formats; colour modes; dtype distribution.Confirms whether the dataset is readable and whether mixed formats or corrupt files may affect training.
spatialHeight, width, aspect ratio, megapixels, file size, and orientation distribution.Shows whether resizing/cropping is needed and whether the dataset has inconsistent resolution.
pixel_statsRGB means, RGB standard deviations, and per-image mean statistics.Supports normalization and reveals channel imbalance or unusual image intensity distributions.
qualityBrightness, contrast, sharpness, noise, entropy, exposure, compression, and blur counts.Helps identify poor-quality images that may harm model learning.
colourSaturation, colour temperature, and grayscale-like counts.Reveals colour bias, warm/cool dominance, and whether images contain enough colour information.
textureGLCM contrast, dissimilarity, homogeneity, energy, correlation, and ASM.Useful for datasets where texture carries class information, such as cracks, surfaces, medical scans, or fabric.
frequencyLow, mid, and high FFT frequency-band energy.Shows whether images are dominated by smooth structures or fine details/noise.
duplicatesExact duplicate groups and near-duplicate groups.Helps prevent data leakage between training, validation, and test sets.
labelsClass distribution and class imbalance ratio.Shows whether sampling, weighting, or augmentation may be needed.

Interpreting the cat and dog dataset outputs

Full dashboard: This combines dataset overview, class balance, image size distributions, quality metrics, colour saturation, entropy, texture, frequency energy, colour temperature, and file format distribution. In the cat/dog example, the labels are balanced, which is important because model training will not be biased toward one class.

Colour analysis: Use this to understand whether your dataset has a strong colour bias. The dominant-colour palette summarizes common pixel colours, while the hue and saturation plots show whether images contain diverse colour information or are concentrated in a narrow colour range.

Quality analysis: Sharpness and noise indicate whether images are clear enough for training. The blurry detection chart separates images below the configured blur threshold. The exposure map helps detect images that are too dark or too bright.

Texture and frequency: GLCM features describe local texture patterns. High contrast suggests stronger texture differences; high homogeneity means neighbouring pixels are more similar. These features are useful when the dataset contains surface patterns, damage marks, or fine visual details.

Channel correlation: The RGB channel correlation matrix shows how related the channels are. Strong correlations are common in natural images, but unusual channel behaviour may suggest colour preprocessing issues or dataset-specific lighting bias.

Common usage pattern for ML projects

# 1. Inspect the dataset
eda.load("dataset/train", label_from_parent=True)
summary = eda.summary()

# 2. Check class balance
print(summary["labels"]["label_distribution"])
print(summary["labels"]["class_imbalance_ratio"])

# 3. Check image quality
print(summary["quality"]["blurry_count"])
print(summary["quality"]["overexposed_frac"])
print(summary["quality"]["underexposed_frac"])

# 4. Get normalization values for model training
norm = eda.normalization_stats()
print("mean:", norm["mean"])
print("std:", norm["std"])

# 5. Export shareable outputs
eda.plot_colour(save_path="colour_dashboard.png")
eda.plot_texture(save_path="texture_dashboard.png")
eda.report("eda_report.html")
